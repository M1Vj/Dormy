## 3. Backend Spec (For Backend Coder)
- **Bootstrap Data (`supabase/seed.sql`)**:
    - Insert 1 `dorms` row (e.g., 'Molave').
    - Insert 1 `profiles` row for the admin.
    - Insert 1 `dorm_memberships` row linking admin to Molave with `role = 'admin'`.
    - *Note*: The Auth User (identity) must be created manually or via a script, but the public tables can be seeded.
- **Middleware (`src/middleware.ts`)**:
    - Refresh session (already there).
    - **Constraint**: If no session, redirect `/(app)` -> `/login`.
    - **Constraint**: If session exists, redirect `/login` -> `/`.
- **Auth Actions (`src/app/actions/auth.ts`)**:
    - `login(formData)`: Validate email/password. Set cookies. Redirect to `/`.
    - `logout()`: Sign out. Remove cookies. Redirect to `/login`.
- **Admin Actions (`src/app/actions/admin.ts`)**:
    - **Constraint**: MUST use `createClient(..., { supabaseKey: process.env.SUPABASE_SERVICE_ROLE_KEY })` to bypass RLS and use `auth.admin` API.
    - `createUser(data)`:
        - Input: `email`, `password`, `firstName`, `lastName`, `role`, `dormId`.
        - Logic:
            1. `supabase.auth.admin.createUser({ email, password, email_confirm: true })`.
            2. Insert into `public.profiles` (id, first_name, last_name).
            3. Insert into `public.dorm_memberships` (user_id, dorm_id, role).
        - Error Handling: Rollback (manual or transaction) if DB insert fails after Auth create? (For MVP, just error out).
        - Security Check: Verify *current user* is an 'admin' before executing.

## 4. Frontend Spec (For Frontend Coder)
- **Login Page (`src/app/(auth)/login/page.tsx`)**:
    - Simple Card centered.
    - Email/Password inputs.
    - "Sign In" button (pending state).
    - Uses `login` server action.
- **Auth Provider (`src/components/providers/auth-provider.tsx`)**:
    - Client Component.
    - Fetches `user` and `profile/membership` on mount.
    - Exposes `user`, `role`, `isLoading`.
- **Sidebar (`src/components/nav/app-sidebar.tsx`)**:
    - Consume `useAuth`.
    - Filter `navMain` items based on `role`.
    - Admin: See everything.
    - Occupant: See Payments, Fines, Evaluation.
- **Admin User Management (`src/app/(app)/admin/users/page.tsx`)**:
    - Table of users (fetch from `profiles` join `dorm_memberships`).
    - "Create User" Dialog.
    - Form: First Name, Last Name, Email, Temp Password, Role (Select), Dorm (Select).
    - Uses `createUser` server action.
