## 3. Backend Spec (For Backend Coder)

### A. Environment & Clients
1.  **Dependencies**: Install `@supabase/supabase-js` and `@supabase/ssr`.
2.  **Clients**:
    - `src/lib/supabase/client.ts`: Browser client using `createBrowserClient`.
    - `src/lib/supabase/server.ts`: Server client using `createServerClient` with cookie handling (Next.js 15+ compatible).
    - `src/middleware.ts`: Middleware to refresh session (`updateSession`) using `createServerClient`.

### B. Schema (`supabase/migrations/20251228000000_init_schema.sql`)
Define the following tables. Ensure all UUIDs default to `gen_random_uuid()`.
**Enable RLS on ALL tables immediately (`ALTER TABLE x ENABLE ROW LEVEL SECURITY;`).**

1.  **Enums**:
    - `app_role`: `admin`, `student_assistant`, `treasurer`, `adviser`, `assistant_adviser`, `occupant`, `event_officer`
    - `occupant_status`: `active`, `left`, `removed`
    - `fine_severity`: `minor`, `major`, `severe`
    - `ledger_category`: `adviser_maintenance`, `sa_fines`, `treasurer_events`
    - `entry_type`: `charge`, `payment`, `adjustment`, `refund`

2.  **Core Tables**:
    - `dorms`: `id` (uuid), `slug` (text unique), `name`, `attributes` (jsonb).
    - `profiles`: `user_id` (uuid, references `auth.users`), `display_name` (text), `avatar_url` (text).
    - `dorm_memberships`: `id`, `dorm_id` (ref `dorms`), `user_id` (ref `profiles.user_id`), `role` (enum `app_role`). Unique(`dorm_id`, `user_id`).
    - `rooms`: `id`, `dorm_id`, `code` (text), `level` (int), `capacity` (int). Unique(`dorm_id`, `code`).
    - `occupants`: `id`, `dorm_id`, `user_id` (nullable ref `profiles`), `full_name`, `student_id`, `status` (`occupant_status`), `joined_at`. Unique(`dorm_id`, `student_id`).
    - `room_assignments`: `id`, `dorm_id`, `room_id`, `occupant_id`, `start_date`, `end_date`.

3.  **Module Tables** (Include `dorm_id` FK on ALL):
    - **Fines**: `fine_rules` (definitions), `fines` (instances).
    - **Ledger**: `ledger_entries` (financials).
    - **Events**: `events`, `event_photos`, `event_ratings` (1-5 stars), `event_teams`, `event_team_members`, `event_scores`.
    - **Evaluation**: `evaluation_cycles`, `evaluation_templates`, `evaluation_metrics`, `evaluation_submissions`, `evaluation_metric_scores`.
    - **Cleaning**: `cleaning_areas`, `cleaning_weeks`, `cleaning_assignments`, `cleaning_exceptions`.
    - **Audit**: `audit_events`.

### C. RLS Policies (`supabase/migrations/20251228000001_rls_policies.sql`)
1.  **Helper Functions** (Security definer):
    - `is_dorm_member(dorm_id uuid)`: Returns bool if `auth.uid()` is in `dorm_memberships` for that dorm.
    - `has_role(dorm_id uuid, roles app_role[])`: Returns bool if user has one of the roles.

2.  **Policies**:
    - **General Rule**: `dorms` table is readable by members.
    - **Strict Isolation**: Users can ONLY read/write data where `dorm_id` matches a dorm they are a member of.
    - **Occupant Privacy**: `occupants` table readable by staff + self.
    - **Feature Access**:
        - `fines`: Read (Own + Staff), Write (Staff only: `admin`, `adviser`, `student_assistant`).
        - `ledger_entries`: Read (Own + Staff), Write (Staff only: `treasurer`, `admin`, `adviser`).

### D. Seed Data (`supabase/seed.sql`)
Insert "Molave Men's Hall" data:
1.  **Dorm**: Slug `molave-mens-hall`.
2.  **Rooms**: `1, 2, 3` (Level 1); `4a, 4b, 5, 6` (Level 2); `7, 8, 9, 10a, 10b` (Level 3).
3.  **Rules**: "Late Return" (Minor), "Vandalism" (Major).
